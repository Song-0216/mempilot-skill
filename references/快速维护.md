# 快速维护与健康报告

## 范围

优先检查上次维护 Git 基线到当前 HEAD 的变化，不全量重读项目。

检查：当前状态是否过期、会话交接是否矛盾、语义路标是否有效、链接与代码路径是否失效、最近代码变化是否漏写、文件是否膨胀、重复事实、过期快照、Agent 入口和维护日期。

## 自动可修复

- 明显失效链接和代码路径
- 当前状态中的已完成旧任务
- 失效或缺失的路标关键词
- 重复短句
- 维护日期和 Git 基线
- 明显低频的旧 Bug 归档

## 先报告再处理

新增、拆分、合并、重命名模块；删除记忆文件；大规模改写；推翻 ADR；大量清理历史。

## 健康报告模式

用户只要报告时，禁止修改。输出：问题、证据、严重度、建议动作和健康状态。
