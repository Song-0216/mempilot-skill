# 日常增量

## 证据

先读取当前状态。

### 有 Git 时

检查：用户任务、`git status`、`git diff`、新增删除移动文件、路由、数据模型、接口、依赖、配置和测试变化。只分析本次变更，不重新扫描整个项目。

### 没有 Git 时

按以下优先级确定变更范围：

1. 本次会话中实际读取、创建、修改和删除的文件记录。
2. 用户任务、当前状态和会话交接。
3. 当前任务涉及的代码路径和模块。
4. 工具支持时参考最近修改文件或文件时间，但不得仅凭时间判断业务含义。

如果仍无法确认：标记“变更范围低置信”，询问用户主要修改了哪些文件或功能，或对当前相关模块执行定向一致性检查。禁止因没有 Git 而全量扫描项目。

非 Git 项目在 `项目索引.md` 维护：Git 基线为“无可用提交”、最近维护日期、最近检查的模块或代码路径。

## 路由与读取

使用项目索引语义路标选择 Top 1 模块，并只读相关 `##` 章节。调试任务再读调试记忆的匹配规则或案例。

## 三问写入

1. 状态变化：局部更新当前阶段、进度、阻塞、下一步或会话交接。
2. 长期认知变化：只更新一个相关模块章节、契约或 ADR。
3. 可复用难题：只更新调试规则、索引或案例。

没有长期价值则不写。

## 局部编辑

- 先读取待修改章节。
- 有 Git 时根据 `git diff`，无 Git 时根据本次会话文件变更形成最小变更块。
- 使用 patch 或精确替换。
- 编辑后检查记忆 diff 或目标章节，避免误删无关内容。
- 普通样式、文案、格式化、局部重命名和一次性调试不写记忆。
