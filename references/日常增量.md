# 日常增量

## 证据

先读取当前状态，然后检查：用户任务、`git status`、`git diff`、新增删除移动文件、路由、数据模型、接口、依赖、配置和测试变化。

只分析本次变更，不重新扫描整个项目。

## 路由与读取

使用项目索引语义路标选择 Top 1 模块，并只读相关 `##` 章节。调试任务再读调试记忆的匹配规则或案例。

## 三问写入

1. 状态变化：局部更新当前阶段、进度、阻塞、下一步或会话交接。
2. 长期认知变化：只更新一个相关模块章节、契约或 ADR。
3. 可复用难题：只更新调试规则、索引或案例。

没有长期价值则不写。

## 局部编辑

- 先读取待修改章节。
- 根据 `git diff` 形成最小变更块。
- 使用 patch 或精确替换。
- 编辑后检查记忆 diff，避免误删无关内容。
- 普通样式、文案、格式化、局部重命名和一次性调试不写记忆。
