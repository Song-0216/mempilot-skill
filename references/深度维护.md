# 深度维护

适用于版本、里程碑、大重构、技术栈变化、主要业务域增删、连续路由失败或记忆明显膨胀。

## 流程

1. 读取维护状态和 Git 基线。
2. 分析基线到 HEAD 的主要变化。
3. 复核业务模块和稳定技术子系统边界。
4. 检查重复、冲突、过期和失效引用。
5. 更新语义路标。
6. 压缩模块和调试记忆。
7. 归档低频历史。
8. 必要时读取 `references/版本与压缩.md` 生成版本记录。
9. 检查 Agent 入口。
10. 更新维护状态和 Git 基线。

## 破坏性操作

模块新增、拆分、合并、重命名或删除前，先输出变更计划：理由、受影响文件、迁移和回滚方式。用户确认后执行，除非用户已明确授权自动整理。
