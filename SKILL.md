# MemPilot

## 目标

维护一套供未来 AI 会话按需读取的中文项目记忆，使长期开发、调试、重构和发布保持连续性，同时控制 Token、上下文污染和人工维护成本。

> 只读当前任务需要的最少记忆，只写未来仍有价值的最小变化。

## 渐进式披露

本文件只负责模式路由和全局硬规则。确定模式后，**只读取一个对应的 `references/*.md`**；除非该 reference 明确要求，否则禁止一次性读取全部 references。

| 模式 | 正式触发语与口语别名 | 读取 reference |
|---|---|---|
| 初始化 | 初始化项目记忆、帮我建记忆 | `references/初始化.md` |
| 日常增量 | 更新项目记忆、记一下、保存进度 | `references/日常增量.md` |
| 快速维护 | 快速维护、检查一下、看看记忆 | `references/快速维护.md` |
| 深度维护 | 深度整理、大扫除、整理归档 | `references/深度维护.md` |
| 一致性检查 | 检查记忆和代码是否一致 | `references/一致性与路标.md` |
| 路标重建 | 重建路标、它又搞错模块了 | `references/一致性与路标.md` |
| 压缩 | 压缩记忆、太费钱了、瘦身 | `references/版本与压缩.md` |
| 版本记录 | 生成版本记录、整理里程碑 | `references/版本与压缩.md` |
| 结构调整 | 新增、拆分、合并、重命名模块 | `references/结构演化.md` |
| Agent 入口 | 切换 Agent、生成入口 | `references/Agent入口.md` |
| 健康报告 | 查看记忆健康状态 | `references/快速维护.md`，只报告不修改 |

路由规则：

1. 没有 `项目记忆/` 时，进入初始化。
2. 有本次代码改动且用户说“记一下”，进入日常增量。
3. “检查一下”默认快速维护，不自动升级为深度维护。
4. 破坏性结构变化必须读取 `references/结构演化.md` 并先给方案。
5. 用户只要报告时不得修改文件。
6. 用户接受维护提议后，直接进入相应模式，不要求重新输入正式命令。

## 固定记忆骨架

```text
项目记忆/
├── 当前状态.md
├── 项目索引.md
├── 调试记忆.md
├── 接口契约.md          # 确有需要时才创建
├── 模块/                # 文件名由真实项目生成
├── 决策/
├── 版本记录/
└── 归档/
```

不要预置示例业务模块。模块按真实业务边界或稳定技术子系统生成，不照抄 `views/`、`components/`、`utils/` 等代码目录。

## Agent 原生入口

只为当前正在使用的工具生成或更新原生入口：

- OpenAI Codex / GPT 编程 Agent：`AGENTS.md`
- Claude Code：`CLAUDE.md`
- Cursor：`.cursor/rules/项目记忆.mdc`
- 其他工具：优先用其原生项目规则；无法确认时用 `AGENTS.md` 并标记待确认

入口必须短，只说明：先读当前状态、按路标选择模块、只读目标章节、默认不读历史、用三问法局部写回。

## 全局读取硬规则

1. 新会话默认只读当前 Agent 入口和 `项目记忆/当前状态.md`。
2. 从任务提取动词、业务名词、接口、字段、错误码和代码路径。
3. 搜索 `项目索引.md` 的模块语义路标，明显命中时只选 Top 1 模块。
4. 明确跨模块或第一模块低置信时才读第二个；普通任务最多 2 个，复杂任务最多 3 个。
5. 模块文件按稳定的 `##` 标题组织；只读匹配章节，不默认全文读取。
6. 默认不读全部 ADR、全部 Bug、全部版本记录、`归档/`、完整 Git 历史或整个代码库。
7. 当前状态或代码路径已明确模块时，可跳过项目索引。

建议预算：入口 300～600 Tokens；当前状态 600～1000；单次路标 100～300；单个模块章节 300～900；普通任务总记忆 1500～3500；复杂任务 3500～7000。

## 全局写入硬规则

任务结束只问三件事：

1. 当前任务状态变了吗？→ 局部更新 `当前状态.md`
2. 长期有效认知变了吗？→ 更新一个模块章节、契约或 ADR
3. 是否解决了可复用难题？→ 更新 `调试记忆.md`

三个答案都是否时，不写记忆。

更新已有文件时必须优先使用 patch、search/replace、精确章节替换或等价的最小差异编辑。除非文件结构重构、模块拆合、文件损坏或工具不支持局部编辑，否则禁止全量重写超过 500 个汉字的已有记忆文件。

写入预算：会话交接最多 5 行；当前进度和下一步各最多 3 条；模块单章节单次更新通常不超过 10 行；ADR 背景不超过 200 字，其余正文不超过 400 字；单个 Bug 案例通常不超过 400 字；高频避坑尽量每条一句话。

## 中文与可信规则

- 目录、文件名、标题、正文和报告使用中文。
- 变量、类、函数、API、字段、配置、框架、错误原文和 `ADR/BUG` 编号保留英文。
- 不确定信息标记“待确认”或“待验证”，不得编造。
- 不保存密码、密钥、Token、连接串、PEM 私钥或其他敏感凭据。
- 同一事实只在一个权威位置保存正文；索引只做导航和语义路标。
- 代码与记忆冲突时，先确认代码是否为当前有效实现，冲突必须显式报告。

## 主动维护提议

达到以下任一情况时，不能只输出模糊的“建议维护”：

- `当前状态.md` 超过约 1000 Tokens
- 单模块超过约 2200 Tokens
- 调试记忆活跃案例超过约 30 个
- 模块超过约 15 个
- 普通任务需要读取 3 个以上模块
- 模块路由连续失败
- 发现重复事实或失效代码路径
- 超过约 30 天未快速维护
- 完成大重构、迁移或版本发布

必须说明：命中了哪些信号、可能影响、建议快速维护还是深度维护，并询问是否现在执行。用户说“好”后直接执行。

- 快速维护：确认一次后可执行非破坏性修复。
- 深度维护：仍需先展示结构变更方案，再单独确认。
- 同一未变化问题在短期内不要反复提醒。

## 完成报告

保持简短：列出已修改文件、待确认项、健康状态和一句下次建议。用户没有追问时，不复述整套方法，不输出长篇解释。
