# Changelog

本项目遵循语义化版本思路。Alpha 阶段可能出现不兼容调整。

## [0.1.2-alpha] - 2026-07-21

### Improved

- 快速维护新增每模块最多 3 条高影响语义陈述的定向漂移抽查
- 语义检查结果统一为“已验证 / 疑似漂移 / 证据不足”，避免静默覆盖
- 同一维护信号使用指纹并默认抑制 7 天，定义提前再次提醒条件
- 项目索引最多保存 5 条未解决提醒状态，解决后删除
- 新增模型能力分级和低置信降级模式
- 初始化只在证据充分时创建模块，并增加模块质量闸门
- 新增语义漂移、提醒抑制、严重度变化和轻量模型回归用例
- 评测指标增加语义漂移误报、重复提醒和低置信产出质量

## [0.1.1-alpha] - 2026-07-21

### Improved

- 维护阈值命中后改为“带证据的主动提议 + 用户一键确认”
- 初始化扫描新增逐步产出、项目类型示例和明确停止条件
- 模块模板增加逐章节填写指导、深度限制和禁写说明
- 日常增量与快速维护增加完整的非 Git 降级路径
- 快速维护增加 1～3 个模块、每模块 3～5 个代码路径的最小腐烂抽查
- 一致性检查增加路径、API、字段、入口和依赖的操作级检查顺序
- 新增主动维护、非 Git 和路径腐烂回归用例

## [0.1.0-alpha] - 2026-07-21

### Added

- 中文优先的项目记忆骨架
- 根据真实项目生成业务模块
- Agent 原生入口适配
- Top 1 模块语义路由
- Markdown 章节级读取
- Patch / Search-Replace 增量写入
- 三问写入法与写入预算
- 日常增量、快速维护和深度维护模式
- 一致性检查、路标重建、压缩和版本记录
- Git 基线与维护状态
- 渐进式 `references/` 与 `assets/` 结构
- Vue 管理项目示例
- Codex、Claude Code 和 Cursor 的原生入口示例
- 触发、模块路由和维护行为的最小评测用例
- README 中的多 Agent 使用与回退说明
- 明确的首发作者版权归属
