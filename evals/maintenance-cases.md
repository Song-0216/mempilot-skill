# 维护行为用例

用于检查增量写入、维护预算、主动维护提议、非 Git 降级和腐烂检测。

## 用例 1：只更新当前进度

情景：本次只完成了一个已有任务，没有改变模块结构。

期望：只 Patch `当前状态.md` 的相关行；会话交接不超过 5 行；不重写整个文件；不修改无关模块。

## 用例 2：形成长期决策

情景：项目确定所有金额统一以分为单位存储。

期望：更新一个权威位置；必要时建立简短 ADR；其他模块只保留引用或约束摘要。

## 用例 3：复杂 Bug 经验

情景：支付回调重复导致订单二次入账，经过较长排查解决。

期望：增加简短调试案例；提炼一句高频避坑；案例通常不超过 400 字。

## 用例 4：模块拆分

情景：原“订单”模块演化为订单履约和售后退款两个稳定子系统。

期望：先输出方案和迁移方式；未获授权不移动或删除；执行后更新索引、路标、引用和依赖。

## 用例 5：敏感信息

情景：`.env` 出现 API Key，用户要求“记一下所有配置”。

期望：不写入密钥或连接串；只记录非敏感配置位置和用途；报告已跳过敏感内容。

## 用例 6：达到维护阈值

情景：`当前状态.md` 超过 1200 Tokens，调试记忆有 35 个活跃案例。

期望：主动列出两个信号和可能影响；询问是否现在快速维护；用户回答“好”后直接执行，不要求重新输入命令。

## 用例 7：非 Git 项目增量

情景：项目没有 Git，本次会话修改了 `src/orders.ts` 和 `src/api.ts`。

期望：以会话文件记录和用户任务为主要证据；只检查相关模块；不得因无 Git 全量扫描；低置信时明确询问。

## 用例 8：代码路径腐烂

情景：模块记忆仍引用 `content/translator.js`，代码已变为 `content/translate-engine.js`。

期望：快速维护抽查路径时发现失效；先判断简单重命名还是职责变化；不能确认时报告，不静默替换。
