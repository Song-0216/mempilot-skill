# 触发用例

用于检查 MemPilot 是否在正确场景进入正确模式。每个用例建议在同一 Agent 上重复运行至少 3 次。

## 应触发

| 用户表达 | 期望模式 | 期望行为 |
|---|---|---|
| 帮我建记忆 | 初始化 | 轻量扫描项目并创建中文项目记忆 |
| 记一下 | 日常增量 | 根据本次代码差异局部更新记忆 |
| 保存进度 | 日常增量 | 更新当前状态与必要长期认知 |
| 检查一下 | 快速维护 | 检查状态、路标、链接与明显过期项 |
| 看看记忆 | 快速维护 | 进行轻量体检，不自动深度重构 |
| 大扫除 | 深度维护 | 先报告破坏性结构方案，再按授权执行 |
| 整理归档 | 深度维护 | 压缩当前记忆并归档低频历史 |
| 它又搞错模块了 | 路标重建 | 检查并更新模块触发关键词 |
| 太费钱了 | 压缩 | 缩小文件和上下文开销 |
| 生成版本记录 | 版本记录 | 根据 Git、ADR、调试和模块状态生成摘要 |

## 不应主动进入记忆维护

以下普通开发请求不应自动运行完整 MemPilot 维护流程：

- 帮我把按钮改成蓝色
- 解释这段 SQL
- 修复这个 TypeScript 类型错误
- 给这个函数补单元测试
- 重构这一个局部组件

完成普通开发后，只有用户明确说“记一下”，或任务产生必须记录的重大长期变化时，才进入增量写入。
