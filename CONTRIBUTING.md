# 贡献指南

感谢你帮助改进 MemPilot。

## 最有价值的贡献

- 不同 Coding Agent 的真实测试结果
- Skill 触发失败或误触发案例
- 模块语义路由选错模块的案例
- 全文读取或整文件重写的违规案例
- 使用 MemPilot 前后的 Token 对比
- 长期开发中记忆失真、膨胀或重复的案例
- 中文文档、模板和示例改进

## 提交 Issue 前

请尽量提供：

1. 使用的 Agent 和版本
2. 项目类型与主要技术栈
3. 用户输入的原始命令
4. 期望行为与实际行为
5. Agent 读取了哪些记忆文件或章节
6. 可公开的最小复现项目或目录结构
7. 是否使用 Git，以及相关差异范围

请勿提交密码、密钥、私有代码或客户数据。

## Pull Request

- 一个 PR 尽量只解决一个问题
- 保持 `SKILL.md` 简短，详细流程优先放入 `references/`
- 新模板放入 `assets/`
- 新增规则必须说明它解决的实际问题，以及是否会增加固定 Token
- 修改模式路由时同步更新 README 和相关示例
- 中文文档使用清晰、直接、可执行的表达

## 设计约束

MemPilot 不计划默认引入：

- 数据库或向量数据库
- 后台服务
- 自动捕获全部聊天
- 强制的复杂开发流程
- 与项目记忆无关的通用 Coding Agent 功能

新功能应至少满足一项：

- 提高记忆准确性
- 降低读取 Token
- 降低写入 Token
- 降低人工维护成本
- 提高长期可读性与可审计性
